Method Description
destroy It will backup mail and subscribe queues so that when Servlet initialization we can reload and continue with existing queue elements.
getIsItWorkingMessage
init loads queues from XML files, to resend them again. Mail Queue has failed mail request id and its retry count. Subscribe Queue has subscribe failed mails and its retry count.
initializeQueues Uncompleted mail and subscribe queues will reloaded from its XML files.
resumeMailing Resumes resending mails if any in mail queue, which will be invoked when there is mail send request or servlet initialize phase. And mails will be tries to resend max 5 times only, if it fails still it will not entertain any more and removes from queue as well as deletes its backup xml file.
resumeSubscribeT_GPS_MAILS_SENT Resumes subscribing mails if any in subscribe queue, which will be invoked when there is mail send request or servlet initialize phase. And mails will be tries to subscribe T_GPS_MAILS_SENT maximum 5 times only, if it fails still it will not entertain any more and removes from queue.
service All mail send requests will be handled in TrinitiMailServlet and This is method where each request will be registered with id and sends them for mailing. if it fails either sending mail or subscribing T_GPS_MAILS_SENT it will writes in XML files for resending/retry purpose. And adds failed mail id to their respective queues.
